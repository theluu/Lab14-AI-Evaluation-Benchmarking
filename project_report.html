5 Câu chuyện kỹ thuật nổi bật

🎣 Bắt quả tang Agent bịa (Hallucination)

Thiết kế câu hỏi "bẫy": đúng chủ đề IT nhưng chi tiết KHÔNG có trong tài liệu. Agent ẩu sẽ điền số bịa; agent có guardrail sẽ từ chối. Ví dụ thật (case GS-052):

V1 — không guardrail · faithfulness 0.0 🟥
"VPN cho tối đa bao nhiêu thiết bị?"
“VPN client GlobalConnect cho phép đăng nhập tối đa trên 3 thiết bị…”
BỊA — tài liệu không hề có con số này
V2 — có guardrail · faithfulness 1.0 🟩
"VPN cho tối đa bao nhiêu thiết bị?"
“Tài liệu không cung cấp thông tin về số lượng thiết bị… vui lòng liên hệ IT Helpdesk 1234.”
BÁM CONTEXT — từ chối đúng
Bài học (Technical Depth) Model nền gpt-4o-mini vốn đã khá "cứng" → guardrail prompt chỉ tạo khác biệt rõ ở input hiểm. Hệ thống eval lượng hóa được khác biệt đó qua chỉ số Faithfulness.

🔁 Vòng lặp Evaluation thực chiến

  • Lần đầu: "bản tối ưu" V2 lại BLOCK — eval bắt được V2 thụt lùi.
  • 5-Whys → root cause: prompt V2 (1) đóng cứng câu từ chối sai bộ phận, (2) từ chối nhầm cả câu trả lời được.
  • Sửa prompt V2 → định tuyến đúng bộ phận (HR vs IT) + cho phép giải thích từ context.
  • Phát hiện nhiễu LLM-judge: Δ dao động ±0.1 giữa các lần chạy → thêm dung sai gate + dùng pass_rate phụ trợ.
  • Thêm 5 bẫy hallucination để tạo khoảng cách điểm thật → V2 thắng +0.118 ổn định → APPROVE.
